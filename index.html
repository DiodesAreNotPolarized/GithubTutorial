Getting Started with Git and GitHub

Installing Git

First, you're going to have to download git from here. Just choose the operating system you have, and install git, following the instructions in the installer. You should also create an account at GitHub, and make a new (empty) Repository.

Setting Everyting Up

Creating the Repo

If you already have code, you're going to want to directly create a git repository with it. Right click the parent folder with all your code, and then click the "Git Bash Here". A command window should appear.

Updating the Server-Sided Repository

After you've made changes on your computer, you're going to want to also change the files in the online repository. To do this, there are a few commands you will need to use.
First, you will have to git add -A. This command adds all (-A is short for --all) the changes you've changed to be commited later.
Then, you'll need to commit to your repo with git commit -m "commit message". You should specify what you have changed in this commit.
Finally, you git push -u origin master.

Updating Your Local Repository

If your team members have changed things to the server-sided repository, you're going to want them on your local repository as well. To do this, you just excecute a git pull, which will update your copy to the one on the server. Make sure you do this every time you're going to change code, as you cannot push the code you've changed if it is not in sync with the server-sided copy.
