Getting Started with Git and GitHub

Installing Git

First, you're going to have to download git from here. Just choose the operating system you have, and install git, following the instructions in the installer (you should keep the default settings). You should also create an account at GitHub, and make a new (empty) repository. Your team will only need one repository, but should each have a seperate GitHub account. The owner of the repository should add the other members, by going to Settings → Collaborators, then entering the usernames of your team members.

Setting Everyting Up

First, you should navigate to the folder where all your project code is going to be stored. In the folder, right click, and there should be an option that says "Git Bash Here". If not, then shift+right click, and then click "Open command window here". Type git into the console, and a list of commands should appear. If not, you haven't installed git corrrectly.
If it's working fine, type git init. It shoud give you Initialized empty Git repository in <folder>/.git/.
You should then make sure you're commiting into your GitHub repository you made before. To do this, use the command git remote add origin https://github.com/Username/RepositoryName.git. The .git that you should use should be shown in the screen where you have your empty repository, and should follow the pattern shown in the code sample.

Updating the Remote Repository

After you've made changes on your computer, you're going to want to also change the files in the online repository. To do this, there are a few commands you will need to use.
First, you will have to git add -A. This command adds all (-A is short for --all) the changes you've changed to be commited later.
Then, you'll need to commit to your repo with git commit -m "commit message". You should specify what you have changed in this commit.
Finally, you git push -u origin master to push your commit to the remote repository. The -u, short for --set-upstream, sets the place you always push and pull from.

Updating Your Local Repository

If your team members have changed things to the remote repository, you're going to want them on your local repository as well. To do this, you just excecute a git pull, which will update your copy to the one on the server. Make sure you do this every time you're going to change code, as you cannot push the code you've changed if it is not in sync with the remote copy.
